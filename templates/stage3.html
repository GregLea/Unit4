Object Oriented Programming (OOP)
Object-oriented programming or OOP as the cool kids call it, is a type of programming that is based on the concept that objects can be made that will contain data that can be accessed and used. This gives programmers the ability to create small containers of data that then can be used to pull data points from, instead of having to create one large all incompassing program. This also helps to break up the work load, allowing programmers to create and test the data without having to build a large program from the start. Since these objects can interact with each other, it is easy to build the pieces of the whole before making the whole. These objects contain classes, instances, methods, modules, and libraries.
Class
While the object is the structure that contains all the data that the programmers wants, classes define the data, sort of acting like a blueprint. For example, you can have a blueprint of a bicycle to make many different types of bikes. In programming, that blueprint is the class.
Instance
If you were to make a red bike and a blue bike, both would be made from the blueprint or Class of Bicylce, but each is a different color. That means that each color bike is an instance of the class bicycle. An instance is the individual example of that class, or the bike made from the blueprint. Since the class defines the data that the program needs to be concerned with, when an instance of that class is run, the code shouldn't be able to access other programs data accidentally, hopefully making the program cleaner and easier to use and debug. OOP also allows the concept of inheritance to work, so that each instance of the class cen be unique, but still inherit qualities from the main class, like having 2 wheels. OOP Inheritance allows programmers to reuse code easily.
Inheritance is when an object or class is based on another existing object or class, and uses some of the same implementations to maintain the same behavior.
Method
Once all of the objects are made, we need to do something with them. In OOP, method is similar to making a procedure that we learned before, basically it is a programmed procedure that is defined as part of the class in any object. There can be more than one method within an object, and that method will only be able to access data that is known by or in that object. Some classes, or subclasses, can override exisitng methods in the main class by using a method with the same name or parameters. Which method that is used is determined by the object that accesses it. This is Method Overriding.
Method overriding is the feature in OOP that allows a subclass or child class to provide a specific implementation of a method that is already provided by one of its parent classes. The implementation in the subclass overrides or replaces the implementation in the parent class by using a method that has same name, parameters, and return as the method in the parent class.
Modules
OOP has the ability to group methods or proceedures in to modules to help with organization. Modules are created so that code in one module doesn't accidentally use the same proceedure of a different module. These modules are usually not linked together until the final program is created and they are linked together then. This allows prograamers to create new modules as needed without having to alter any existing modules.
Library
A library in OOP is the collection of all the prewritten classes, any of which the programmer can access and use. The programmer specifies which classes or objects to use when the program is executed. For example, I could have three classess for the steps in my web notes in my library, and the webpage program would pull from that libary in the order that I specify when creating the webpage.
HTML, CSS and OOP

This example is to help me try to grasp the idea of abstraction. Abstraction has been defined to me as the process of taking a real world object and trying to recreate that in programming terms. By breaking down the object bike in to this data, I can see how to make that into data that can be used as a blueprint. Abstraction allows me and a programmer to focus on the details that are important to the final product. If I need to make many bikes, I need to know what makes up a bike. Abstraction seems to be the concept of moving the focus from the details and concrete implementation of things, to the types of things (classes), the operations available (methods), to making the programming simpler, more general, and more abstract.
